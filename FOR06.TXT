
	Обмены с накопителем на магнитной ленте

--------------------------------------------------------------
  Обращение               Результат
--------------------------------------------------------------

  FSAVE		создается копия Форта на ленте. Все
		значения переменных и скомпилированная про-
		грамма сохраняются в копии

 CLOAD <fname>	производится поиск на ленте текстового файла
		с последующим чтением и компиляцией. Для 
		выхода в режим компиляции достаточно нажатия
		любой клавиши

  <X> <L/S>	открывается файл, имя которого должно быть пред-
   COPEN	варительно занесено в строку - указатель, причем 
		Х - адрес строки указателя, а вершина стека
		L/S - определяет режим чтение/запись :
		при L/S=0 - (чтение) на ленте отыскивается
		заголовок файла с заданным в указателе именем.
		Имена пропускаемых файлов выводятся на монитор ;
		при L/S=1 - (запись) на ленту записывается
		заголовок файла с заданным в указателе именем

 <X> CPUT	пересылка младшего байта с вершины стека в
		буфер записи. После заполения буфера он 
		записывается на ленту и очищается для про-
		должения записи. Незаполненый буфер может
		быть записан на ленту командой CCLOSE

   CGET		<X>  - пересылка байта из буфера чтения
		на вершину стека. Чтение нового блока с ленты
		начинается только после вызова последнего байта
		блока. Буфер чтения очищается командой CCLOSE

   CEOF		<L> - анализируется следующий байт буфера
		чтения. Если он совпадает с признаком 
		конца файла  ( 1А ), то  L=1. Во всех ос-
		тальных случаях  L=0

  CCLOSE	закрытие файла и выключение двигателя магни-
		тофона. Если файл был открыт на запись, записы-
		вается последний блок, дополненный до  100Н
		кодами  1А. Если файл был открыт на чтение,
		то очищается буфер чтения и сбрасывается его
		указатель

------------------------------------------------------------------

		Управляющие слова

------------------------------------------------------------------
  Обращение         Результат
------------------------------------------------------------------
   ABORT	переход к режиму интерпретации со сбросом
		стеков возвратов и операндов

 <N> EXECUTE	передача управления на адрес  N

    BYE		передача управления на адрес 0000, то есть 
		СР/М  или монитору

   STMEM	установка карты распределения памяти в соответ-
		ствии с  СР/М ; переход в режим  25-ти строк

-----------------------------------------------------------------

		Работа со словарем

-----------------------------------------------------------------
  Обращение         Результат
-----------------------------------------------------------------

 HERE		<X> - в стек заносится адрес первого свобод-
		ного байта словаря, то есть вершина словаря

 <X> ,		число с вершины стека записывается в словарь,
		вершина которого сдвигается на два байта
		 ( HERE=HERE+2 )

 <B> C,		байт с вершины стека записывается в словарь,
		вершина которого сдвигается на один байт
		( HERE=HERE+1 )

 <N> ALLOT	резервирование в словаре N - байтов
		( HERE=HERE+N )

 <X> CONSTANT	инициализация константы с именем NAME и
    <NAME>	записью в нее значения X .Имя NAME заносится
		в словарь и становится самостоятельной статьей,
		при обращении к которой в стек вызывается значе-
		ние константы

 <X> VARIABLE	инициализация переменной с именем NAME и
    <NAME>	запись в нее значения X. Имя NAME заносится
		в словарь и становится самостоятельной статьей,
		при обращении к которой в стек вызывается адрес,
		содержащий значение переменной

 <N> STRING	инициализация байтового массива из N - элементов
   <NAME>	имя NAME заносится в словарь и становится са-
		мостоятельной статьей, при обращении к которой 
		<X> NAME в стек заносится адрес Х-го элемента 
		байтового массива

  PAD		<X> - в стек заносится адрес "блокнота",
		то есть начало отнсительно свободной зоны
		памяти ( PAD=HERE+40 )

  TIB		<X> - в стек заносится адрес буфера ввода-
		вывода с клавиатуры

FORGET <NAME>	из словаря выбрасывается статья с введенным
		именем, а также все статьи введенные после нее

  ' <NAME>	<X> - поиск в словаре и запись на вершину
		стека адреса пускача статьи с именем NAME

 <X> FIND	поиск в словаре статьи по строке описателю,
		адрес которой берется из Х. Если слово найдено,
		то в стеке : 
		<Y> <L> - где Y - адрес пускача найденного
		об"екта, а L :
		L = 1  если об"ект компилируемый и
		L =-1  если об"ект  немедленно исполняемый
		Если слово не найдено в словаре, то в стек 
		заносится  <0>

   WORDS	вывод каталога словаря

 IMMEDIATE	последний об"ект словаря превращается в
		немедленно исполняемый

VOCABULARY	определение нового словаря
