
		Организация циклов и ветвлений

------------------------------------------------------------------
  Конструкция	     Алгоритм исполнения
------------------------------------------------------------------

 <MAX> <MIN>	выполняется тело цикла , начиная с  MIN  и
 DO <тело>	кончая  MAX-1, то есть  (MAX-MIN) - раз
    LOOP	Шаг цикла принимается равным  1. Значение управ-
		ляющей переменной внутреннего цикла может быть
		вызвано в стек обращением  I. При вложении 
		циклов внутри цикла можно также вызвать в стек 
		значение управляющей переменной цикла 2-го уров-
		ня обращением  J. При любых  MAX, MIN
		цикл выполняется, как минимум один раз

 <MAX> <MIN>	выполняется аналогично, но шаг цикла STEP,
 DO <тело>	каждый раз берется со стека
 <STEP> +LOOP

  LEAVE		принудительный выход из внутреннего цикла,
		то есть обход ближайшего  LOOP

  <L> IF	в зависимости от логического значения, беруще-
 <тело1>	гося со стека выполняется :
   ELSE		если  L <> 0 ( true ) -  тело1 ;
 <тело2>	если  L = 0 ( false ) -  тело2.
   THEN		Допускаются конструкции :
		IF <тело> THEN
		IF ELSE <тело> THEN

  BEGIN		выполняется тело до тех пор пока логическое
 <тело>		значение на вершине стека не станет ( true ),
 <L> UNTIL	то есть до  L <> 0. При любом  L тело выпол-
		няется, как минимум, один раз

  BEGIN		1. Исполняется тело ;
 <тело>		2. Проверяется L с вершины стека на истинность ;
 <L> WHILE	3. Если  L = 0  ( false ), то управление пере-
 <тело1>	дается следующему за REPEAT  об"екту ;
  REPEAT	4. Если  L <> 0 ( true ), то исполняется тело1 
		и далее в соответствии с п.1

    Описанные конструкции могут использоваться только внутри
		 определений
